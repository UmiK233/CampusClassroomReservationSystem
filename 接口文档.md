# 接口文档

## 1. 文档说明

本文档基于当前后端控制器代码与前端实际调用重新整理，适用于本项目当前版本。

- 后端项目路径：`backend/src/main/java/org/campus/classroom/controller`
- 前端接口封装路径：`frontend/src/api/index.js`
- 前端请求基路径：`/api`

除特别说明外，所有接口响应格式统一为：

```json
{
  "code": 200,
  "message": "success message",
  "data": {}
}
```

导出类接口除外，直接返回文件流。

## 2. 通用约定

### 2.1 Base URL

前端通过代理访问后端，统一前缀为：

```text
/api
```

例如登录接口完整路径为：

```text
/api/auth/login
```

### 2.2 认证方式

登录成功后，后续受保护接口需在请求头中携带：

```text
Authorization: Bearer <token>
```

### 2.3 主要业务状态值

#### 用户角色

- `ADMIN`
- `TEACHER`
- `STUDENT`

#### 教室/座位状态

- `ENABLED`
- `DISABLED`

#### 预约状态

- `ACTIVE`
- `CANCELLED`
- `EXPIRED`

#### 签到状态

- `PENDING`
- `CHECKED_IN`
- `NO_SHOW`
- `CANCELLED`

#### 候补状态

- `WAITING`
- `PROMOTED`
- `CANCELLED`
- `EXPIRED`

#### 资源类型

- `CLASSROOM`
- `SEAT`

### 2.4 通用错误码

- `200` 成功
- `400` 参数错误
- `401` 未登录或认证失败
- `403` 无权限
- `404` 资源不存在
- `409` 业务冲突
- `500` 系统错误

## 3. 鉴权接口

### 3.1 登录

- 方法：`POST`
- 路径：`/auth/login`
- 权限：公开

请求体：

```json
{
  "username": "student01",
  "password": "12345678"
}
```

字段说明：

- `username`: 用户名，必填
- `password`: 密码，必填

响应 `data`：

```json
{
  "token": "jwt-token",
  "userInfo": {
    "id": 10001,
    "username": "student01",
    "nickname": "student01",
    "email": "student01@example.com",
    "role": "STUDENT",
    "creditScore": 100,
    "creditLevel": "A",
    "seatReservationAdvanceHours": 24,
    "maxSingleReservationMinutes": 360,
    "dailyReservationLimitMinutes": 720
  }
}
```

### 3.2 注册

- 方法：`POST`
- 路径：`/auth/register`
- 权限：公开

请求体：

```json
{
  "username": "student01",
  "password": "12345678",
  "email": "student01@example.com"
}
```

字段说明：

- `username`: 必填，长度 `6-20`
- `password`: 必填，长度 `8-20`
- `email`: 必填

### 3.3 获取当前用户信息

- 方法：`GET`
- 路径：`/auth/me`
- 权限：已登录

响应 `data` 字段同登录接口中的 `userInfo`

### 3.4 修改密码

- 方法：`PUT`
- 路径：`/auth/password`
- 权限：已登录

请求体：

```json
{
  "oldPassword": "12345678",
  "newPassword": "newpassword123"
}
```

字段说明：

- `oldPassword`: 旧密码，必填
- `newPassword`: 新密码，必填，长度 `8-20`

## 4. 教室与座位查询接口

### 4.1 获取教室详情

- 方法：`GET`
- 路径：`/classrooms/{id}`
- 权限：已登录可访问

路径参数：

- `id`: 教室 ID

响应 `data`：

- `id`
- `roomNumber`
- `building`
- `seatRows`
- `seatCols`
- `capacity`
- `status`
- `remark`

### 4.2 获取楼栋列表

- 方法：`GET`
- 路径：`/classrooms/buildings`
- 权限：已登录可访问

响应 `data`：

```json
["教学楼A", "教学楼B"]
```

### 4.3 获取教室座位布局

- 方法：`GET`
- 路径：`/classrooms/{id}/seats`
- 权限：已登录可访问

路径参数：

- `id`: 教室 ID

响应 `data`：

- `classroomId`
- `classroomBuilding`
- `classroomNumber`
- `seatRows`
- `seatCols`
- `seatVOS`: 座位数组

`seatVOS` 元素字段：

- `id`
- `classroomId`
- `seatNumber`
- `rowNumber`
- `colNumber`
- `status`
- `remark`

### 4.4 查询可用教室列表

- 方法：`GET`
- 路径：`/classrooms/available_list`
- 权限：已登录可访问

查询参数：

- `building`: 可选，楼栋名称
- `min_capacity`: 必填，最小容量

响应 `data`：

- `ClassroomVO[]`

### 4.5 获取用户楼栋偏好

- 方法：`GET`
- 路径：`/classrooms/preferred_buildings`
- 权限：`STUDENT` / `TEACHER`

响应 `data`：

```json
[
  {
    "building": "教学楼A",
    "reservationCount": 12
  }
]
```

### 4.6 查询某教室某时段已预约座位 ID

- 方法：`GET`
- 路径：`/classrooms/{id}/reserved_seats`
- 权限：`STUDENT` / `TEACHER`

路径参数：

- `id`: 教室 ID

查询参数：

- `start_time`: 必填，ISO 时间，例如 `2026-05-01T08:00:00Z`
- `end_time`: 必填，ISO 时间

响应 `data`：

```json
[101, 102, 103]
```

## 5. 预约接口

### 5.1 创建座位预约

- 方法：`POST`
- 路径：`/reservations/seats`
- 权限：`STUDENT`

请求体：

```json
{
  "seat_id": 101,
  "start_time": "2026-05-01T08:00:00Z",
  "end_time": "2026-05-01T10:00:00Z",
  "reason": "自习"
}
```

字段说明：

- `seat_id`: 座位 ID，必填
- `start_time`: 开始时间，必填
- `end_time`: 结束时间，必填
- `reason`: 原因，可选，最大 255 字符

响应 `data`：

- 新建预约 ID

### 5.2 创建整间教室预约

- 方法：`POST`
- 路径：`/reservations/classrooms`
- 权限：`TEACHER`

请求体：

```json
{
  "classroom_id": 8,
  "start_time": "2026-05-01T08:00:00Z",
  "end_time": "2026-05-01T10:00:00Z",
  "reason": "授课"
}
```

字段说明：

- `classroom_id`: 教室 ID，必填
- `start_time`: 开始时间，必填
- `end_time`: 结束时间，必填
- `reason`: 原因，可选

响应 `data`：

- 新建预约 ID

### 5.3 取消预约

- 方法：`DELETE`
- 路径：`/reservations/{id}`
- 权限：`STUDENT` / `TEACHER`

路径参数：

- `id`: 预约 ID

### 5.4 预约签到

- 方法：`POST`
- 路径：`/reservations/{id}/check-in`
- 权限：`STUDENT`

路径参数：

- `id`: 预约 ID

### 5.5 查询当前有效预约

- 方法：`GET`
- 路径：`/reservations`
- 权限：`STUDENT` / `TEACHER`

响应 `data`：

- `ReservationVO[]`

`ReservationVO` 字段：

- `id`
- `resourceType`
- `resourceId`
- `resourceName`
- `reserveDate`
- `startTime`
- `endTime`
- `reason`
- `status`
- `attendanceStatus`
- `checkInTime`
- `canCheckIn`
- `createTime`

### 5.6 查询历史预约

- 方法：`GET`
- 路径：`/reservations/history`
- 权限：`STUDENT` / `TEACHER`

响应结构同 `GET /reservations`

- `windowDays`
- `windowLabel`
- `classroomCount`
- `enabledClassroomCount`
- `totalUserCount`
- `totalReservations`
- `activeReservations`

- `overallUtilizationRate`

- `userReservationList`

### 8.2 教室管理

#### 创建教室

- 方法：`POST`
- 路径：`/admin/classrooms`

请求体：

```json
{
  "roomNumber": "101",
  "building": "教学楼A",
  "seatRows": 8,
  "seatCols": 6,
  "status": "ENABLED",
  "remark": "多媒体教室"
}
```

#### 更新教室

- 方法：`PUT`
- 路径：`/admin/classrooms/{id}`

路径参数：

- `id`: 教室 ID

请求体同创建教室

#### 初始化座位布局

- 方法：`POST`
- 路径：`/admin/classrooms/{id}/seat_layout`

#### 新增座位

- 方法：`POST`
- 路径：`/admin/classrooms/{id}/seats`

请求体：

```json
{
  "seatNumber": "1-1",
  "rowNumber": 1,
  "colNumber": 1,
  "status": "ENABLED",
  "remark": ""
}
```

#### 批量更新某教室座位状态

- 方法：`PATCH`
- 路径：`/admin/classrooms/{id}/seat_layout`

请求体：

```json
{
  "status": "DISABLED",
  "remark": "维护中"
}
```

说明：

- 当前接口复用 `SeatUpdateDTO`
- 实际使用中主要用于整间教室的座位批量启用/禁用

#### 查询教室列表

- 方法：`GET`
- 路径：`/admin/classrooms/list`

查询参数：

- `building`: 必填
- `min_capacity`: 必填
- `status`: 必填，`ENABLED` / `DISABLED`

#### 删除整间教室座位布局

- 方法：`DELETE`
- 路径：`/admin/classrooms/{id}/seat_layout`
