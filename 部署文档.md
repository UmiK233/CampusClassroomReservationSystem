# 部署文档

## 1. 项目概述

本项目为校园教室预约管理系统，采用前后端分离架构：

- 后端：`Spring Boot 3.5.9`、`Spring Security`、`MyBatis`、`JWT`、`Redis`
- 前端：`Vue 3`、`Vite`、`Pinia`、`Element Plus`、`ECharts`
- 数据库：`MySQL 8`
- 辅助组件：`Redis 7`、`SMTP 邮件服务`、`Nginx`、`Docker Compose`

默认开发端口：

- 前端开发服务器：`5173`
- 后端接口服务：`8080`
- MySQL（Docker）：`3307`
- Redis：`6379`
- Nginx（Docker）：`80`

## 4. MySQL 配置

### 4.1 创建数据库

```sql
CREATE DATABASE classroom DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

建议：

- 字符集使用 `utf8mb4`
- 时区统一为 UTC，业务展示层按北京时间转换

### 4.2 后端连接配置

当前本地开发配置位于 `backend/src/main/resources/application-local.yaml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/classroom?useUnicode=true&characterEncoding=utf8&serverTimezone=UTC&useSSL=false&allowPublicKeyRetrieval=true
    username: root
    password: 123456
    driver-class-name: com.mysql.cj.jdbc.Driver
```

部署前至少需要调整：

- `spring.datasource.url`
- `spring.datasource.username`
- `spring.datasource.password`

## 5. Redis 配置

Redis 主要用于：

- `refreshToken` 存储与轮换
- 邮箱验证码存储与冷却控制
- 登录设备会话管理
- 三设备上限与最旧设备挤下线策略

当前本地开发配置：

```yaml
spring:
  data:
    redis:
      host: localhost
      port: 6379
```

说明：

- 当前项目默认未启用 Redis 密码
- 如果你的 Redis 开启了鉴权，需自行补充 `spring.data.redis.password`

## 6. Mail 配置

邮箱服务用于：

- 注册验证码
- 找回密码验证码
- 验证码登录

当前本地示例配置位于 `application-local.yaml`：

```yaml
spring:
  mail:
    host: smtp.qq.com
    port: 465
    username: your-email@example.com
    password: your-app-password
    properties:
      mail:
        smtp:
          auth: true
          ssl:
            enable: true
app:
  mail:
    from: ${spring.mail.username}
  auth:
    email-code-expiration-seconds: 300
    email-code-resend-seconds: 60
```

部署时需要替换：

- `spring.mail.username`
- `spring.mail.password`
- `app.mail.from`

常见说明：

- QQ 邮箱需要填写“授权码”，不是邮箱登录密码
- 如果使用 163、企业邮箱或 Gmail，需要按对应 SMTP 要求调整端口与 SSL/TLS 配置

## 7. JWT 与认证相关配置

当前配置项：

```yaml
jwt:
  secret: change-this-in-production
  access-expiration: 1800000
  refresh-expiration: 604800000
```

含义：

- `access-expiration=1800000`：access token 默认 30 分钟
- `refresh-expiration=604800000`：refresh token 默认 7 天

建议：

- 生产环境必须替换 `jwt.secret`
- 不要把真实密钥直接提交到公开仓库

## 9. 升级 SQL

如果你使用的是较早版本数据库，而不是重新导入 `classroom.sql`，需要执行升级脚本。

### 9.1 认证升级脚本

文件：

```text
backend/sql/auth_token_upgrade.sql
```

作用：

- 为 `user` 表补充 `token_version` 字段

适用场景：

- 老库还停留在单 token 阶段
- 登录链路已升级为 `accessToken + refreshToken`

### 9.2 管理员审计升级脚本

文件：

```text
backend/sql/admin_audit_log_upgrade.sql
```

作用：

- 新增 `admin_audit_log` 表
- 支持管理员操作审计日志查询

### 9.3 推荐执行顺序

如果是老库升级，推荐顺序：

1. 备份旧数据库
2. 执行 `backend/sql/auth_token_upgrade.sql`
3. 执行 `backend/sql/admin_audit_log_upgrade.sql`
4. 启动后端服务检查应用是否正常连接

## 10. 默认账号

当前 `backend/sql/classroom.sql` 种子数据中包含若干示例账号，便于演示和测试。

建议优先使用：

| 角色 | 用户名 | 备注 |
| --- | --- | --- |
| 管理员 | `admin` | 后台管理、审计、强制下线 |
| 学生 | `student01` | 学生预约、签到、候补演示 |
| 教师 | `233` | 教师整间教室预约演示 |

当前种子数据的命名方式与密码策略保持一致，通常为“用户名即初始密码”，例如：

- `admin / admin`
- `student01 / student01`
- `233 / 233`

如果你修改了 SQL 或登录失败，建议直接在 `backend/sql/classroom.sql` 中替换为你自己的 BCrypt 密码后重新导入。

## 11. 本地启动步骤

- Redis

## 14. 常见问题

### 14.1 登录后接口一直 401

检查：

- Redis 是否启动
- `jwt.secret` 是否一致
- 老库是否已经补齐 `token_version`

### 14.2 邮箱验证码发送失败

检查：

- `spring.mail.username` 与 `spring.mail.password` 是否正确
- 是否使用了 SMTP 授权码
- 服务器是否允许访问对应 SMTP 主机和端口

### 14.3 管理员操作日志报表不存在

检查：

- 是否执行了 `backend/sql/admin_audit_log_upgrade.sql`
- 或者是否直接导入了最新 `backend/sql/classroom.sql`

### 14.4 旧设备无法自动下线

检查：

- Redis 是否正常写入设备会话
- 前端是否带上 `X-Device-Id` 和 `X-Device-Name`
- 是否多个浏览器共用了同一个本地存储环境

## 15. 部署建议

- 生产环境替换所有默认密钥与邮箱配置
- MySQL、Redis 建议启用独立密码与备份策略
- 将 `application-local.yaml` 中的敏感信息改为环境变量注入
- 前端建议由 Nginx 统一托管，后端只暴露 API
- 如果后续持续迭代，建议补充 Flyway 或 Liquibase 管理数据库迁移
