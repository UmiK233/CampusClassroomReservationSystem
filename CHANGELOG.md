# Changelog

## [v0.3.0] - 2026-04-21

### ✨ 新增功能

* 新增每日预约总时长限制

### 🔄 更新

* 单次预约时长上限设置为 3 小时
* 每日预约上限设置为 9 小时
* 增加reservation_usage表记录预约时长和日期

### 🐛 修复

* 修复跨天预约未校验的问题
* 修复并发穿透问题:

同一个用户同时发两个请求：

```
请求A：查 → 当前 6h → 可以预约
请求B：查 → 当前 6h → 也可以预约

A 插入 +3h → 总 9h
B 插入 +3h → 总 12h （超了）
```

   现在后端主要有登录注册、教室查询、座位布局、预约、取消、管理员教室/座位维护。没有分页、统计、用户管理、审批、通知、
   课表、公告、预约详情、日历视图等更丰富的业务接口，所以前端能展示的数据面比较窄。
2. 前端目前偏“功能可用”，不是完整产品
   现在页面是：
    - 登录/注册
    - 教室与座位
    - 我的预约
    - 后台管理

   这些是核心流程，但缺少很多产品化细节，比如首页总览、预约日历、教室详情页、筛选侧栏、状态看板、操作日志、空状态设
   计、帮助提示等。
3. 数据展示方式偏表格
   表格能快速完成管理功能，但视觉上会显得“内容少”。比如教室列表、预约列表、后台管理都主要是表格，缺少日历、时间轴、教
   室卡片、座位热力图、统计图等更丰富的表达。
4. 学生、教师、管理员没有完全分角色做体验
   现在是同一套主页面根据角色显示按钮。更完整的前端应该是：
    - 学生：找座位、预约记录、常用教室、今日预约
    - 教师：整间教室预约、可用教室推荐、预约冲突提示
    - 管理员：资源总览、教室维护、座位维护、预约监控
5. 缺少“围绕场景”的页面
   现在是接口驱动，不是场景驱动。比如用户真正想做的是“今晚 7 点找个能坐的座位”，更好的页面应该先选时间和教学楼，然后直
   接推荐可用教室/座位，而不是先点教室再看座位。

可以补强的方向：

- 做一个首页仪表盘：今日预约、可用教室数、当前占用座位、快捷入口
- 教室页改成卡片 + 表格混合：左侧筛选，右侧教室卡片，点开座位图
- 加预约日历视图：按天/周查看预约
- 加教室详情抽屉：容量、座位状态、备注、预约按钮
- 管理员页拆分：教室管理、座位管理、预约监控
- 加空状态、加载骨架屏、错误状态、权限提示
- 学生端做“按时间找座位”的主流程
- 教师端做“按时间找空教室”的主流程
